# CLAUDE.md — 프로젝트 규약 (Claude Code / 에이전트)

이 파일은 저장소 **최상위**에서 에이전트가 따라야 할 **지도**다. 하위 앱·패키지의 `brain` 문서와 함께 읽는다.

---

## `brain` 폴더가 의미하는 것

**`brain`은 “이 코드베이스(또는 이 앱/패키지)를 어떻게 다룰지”를 사람과 에이전트가 공유하는 지식 저장소**다.

- **역할**: 규칙, 워크플로, 브랜드·목표 같은 **맥락**을 코드 옆에 두어, 에이전트가 매번 전체 코드를 추측하지 않고 **의도된 방식**으로 작업하게 한다.
- **위치**: 루트에 둘 수도 있고, `apps/<앱명>/brain`, `packages/<패키지명>/brain`처럼 **스코프별**로 둔다. 스코프가 좁을수록 그 폴더 안 내용은 **해당 영역 전용**이다.
- **레이어 이름**: 아래 **P0~P6** 표에 맞춰 `p0-trust-first`, `p1-automation-rhythm` 등 **하위 폴더**로 나눈다. 숫자가 **낮을수록 불변에 가깝고**, **높을수록 자주 바뀌는** 내용이다.

요약하면, **`brain` = 이 레포(또는 이 모듈)의 “운영·개발 두뇌”를 md로 고정한 곳**이며, 에이전트는 작업 디렉터리에 맞는 `brain`만 골라 읽으면 된다.

---

## SKILL.md 규약

- **한 파일은 최대 약 100줄**까지를 권장한다. 그 이상이면 에이전트가 해당 `.md`를 **규약**이 아니라 **참고 자료**로만 취급해 준수율이 떨어질 수 있다.
- **100줄을 넘기는 상세 내용**은 같은 주제 폴더의 `references.md` 등으로 분리한다. (ex: fsd rules - ./brain/p0-trust-first/fsd/)
- **SKILL.md에 두기 좋은 내용 (예시)**  
  스킬 개요·사용자 시나리오, 퀵 레퍼런스, 브랜드·아이덴티티 가치, 일반적인 사용 사례

## Brain 레이어 (P0 ~ P6)

| 레이어 | 기능 (Brain)     | 폴더 예시              | 설명                                                                        |
| ------ | ---------------- | ---------------------- | --------------------------------------------------------------------------- |
| **p0** | 생존, 보안, 불변 | `p0-trust-first`       | 변하지 않는 절대 규칙·제약·보안                                             |
| **p1** | 자동화, 반복     | `p1-automation-rhythm` | CI/CD, 포맷, 반복 빌드·배포                                                 |
| **p2** | 기억, 상태       | `p2-state-memory`      | 전역 상태·캐시·세션 등 **데이터 흐름** 명세                                 |
| **p3** | 위험 감지, 방어  | `p3-risk-detect`       | 에러 처리, 테스트 가드레일, 런타임 예외                                     |
| **p4** | 실행, 작업       | `p4-workflow`          | UI·비즈니스 로직 **실행 순서**, 실무 SKILL                                  |
| **p5** | 스타일, 페르소나 | `p5-brand-identity`    | 브랜드 가치, 톤앤매너, 디자인 토큰(Color/Typo)                              |
| **p6** | 목표, 전략       | `p6-goal-strategy`     | 스프린트·로드맵·우선순위 — **가장 자주 갱신**, 에이전트가 자주 읽을 수 있음 |

## 디렉터리 구조 (개념)

루트에 전체 지도를 두고, 상세는 `brain` 하위 레이어로 나눈다.

```
/
├── CLAUDE.md                    # P0~P6 전체 규약·지도
└── brain/
    ├── p0-trust-first/          # [고정] 보안·신뢰
    ├── p1-automation-rhythm/    # [반복] 자동화·CI
    ├── p2-state-memory/         # [저장] 상태·데이터 흐름
    ├── p3-risk-detect/          # [방어] 리스크·테스트
    ├── p4-workflow/             # [실행] 주요 SKILL
    ├── p5-brand-identity/       # [개성] 브랜드·UI 토큰
    └── p6-goal-strategy/        # [가변] 현재 목표·우선순위
```

---

## 루트 vs 앱·패키지

### 루트 (공통)

모든 앱·패키지가 **공유**해야 하는 원칙을 둔다.

- **p0-trust-first**: 보안, 의존성 금기 등
- **p1-automation-rhythm**: 전체 빌드(Turborepo/Nx 등), CI/CD
- **p5-brand-identity**: 브랜드 가치, 공통 디자인 토큰, 말투
- **p6-goal-strategy**: 모노레포 전체 로드맵(필요 시)

### 앱·패키지 (개별)

**그 프로젝트에만 해당**하는 가변 레이어를 둔다.

- **p2-state-memory**: 해당 앱의 상태 트리·데이터 흐름
- **p4-workflow**: 해당 프로젝트의 **SKILL.md** (컴포넌트·로직 작성법)
- **p6-goal-strategy**: 해당 서비스의 단기 목표·우선순위
